## Project Overview

**nuxt-ai-ready** is a Nuxt module that makes websites discoverable by AI agents and LLMs through standardized APIs and protocols. Uses build-time vector indexing with runtime semantic search via vector databases.

Key components:
- **Build-time processing**: HTML → markdown → chunks → embeddings → vector DB during prerender
- **Runtime APIs**: semantic search, RAG chat, bulk JSONL export, MCP server
- **AI discovery**: llms.txt generation, markdown serving, standardized endpoints
- **Vector DB support**: sqlite-vec, libsql, pgvector, Cloudflare Vectorize, Upstash

## Architecture

### Module Entry (`src/module.ts`)

Nuxt module definition with these hooks:
- `ai-ready:llms-txt`: Allow other modules to extend llms.txt content
- `ai-ready:routes`: Allow other modules to register API routes
- `nitro:init`: Build-time processing via `prerender:generate` and `prerender:done` hooks

Build-time flow (via `src/prerender.ts`):
1. `prerender:generate`: Process .md files generated by mdream middleware, store page data
2. `prerender:done`: Generate llms.txt artifacts from collected pages

### Runtime Server (`src/runtime/server/`)

MCP definitions auto-scanned from:
- `mcp/tools/`: MCP tools (get_page, list_pages)
- `mcp/resources/`: MCP resources (pages://list, content://all)
- `mcp/prompts/`: MCP prompts (search-content, explain-concept, find-information)

Uses `defineMcpTool`, `defineMcpResource`, `defineMcpPrompt` from `#mcp-toolkit/utils`.

### Key Dependencies

- **mdream**: HTML → markdown conversion, chunking, llms.txt generation
- **@nuxtjs/mcp-toolkit**: MCP server implementation (dev dependency, optional runtime)
- **nuxt-site-config**: Site metadata (peer dependency)
- **tokenx**: Token counting for chunk sizing
- **minimatch**: Pattern matching for route filtering

### Type Exports

- `ModuleOptions`: Module configuration interface
- `BulkDocument`: JSONL document format (route, title, description, markdown, headings, chunkIds)

## Testing Patterns

### E2E Tests
Use `@nuxt/test-utils` with `setup({ build: true, server: true })`. Tests verify:
1. Build artifacts exist (vector DB, bulk JSONL, debug chunks)
2. Bulk API returns valid JSONL
3. MCP server handles initialize, tools/list, tools/call

### Vitest Projects
- **unit**: Node environment, `./test/unit/**/*.test.ts` + `./src/**/*.test.ts`
- **e2e**: `./test/e2e/**/*.test.ts`

## Module Configuration

Default config in `module.ts`:
```ts
{
  enabled: true,
  debug: false,
  bulk: {
    enabled: true,
    route: '/_ai-ready/bulk',
  },
}
```

Config key: `aiReady` in nuxt.config.ts

Runtime config:
- `nuxt-ai-ready.version`: Module version
- `nuxt-ai-ready.debug`: Debug mode
- `aiSearch.routes`: API routes (bulk, mcp)

## Important Notes

- Module requires Nuxt ≥4.0.0
- Prerendering happens during `nuxi generate` or `nuxi build --prerender`
- Uses `#ai-ready` alias for runtime imports
- MCP paths registered via `mcp:definitions:paths` hook (from @nuxtjs/mcp-toolkit)
- llms.txt generation uses mdream's `generateLlmsTxtArtifacts` function
- Vector IDs generated via SHA256 hash: `generateVectorId(route, chunkIdx)` in `src/module.ts:25`
