Nuxt AI Ready generates `/llms.txt` and `/llms-full.txt` during prerender following the [llms.txt standard](https://llmstxt.org/).

## Generation Process

Build-time flow via prerender hooks:

1. `prerender:generate` - HTML → markdown via mdream, extract metadata
2. Page data collected and stored
3. `prerender:done` - `generateLlmsTxtArtifacts()` writes both files to `.output/public/`

Includes site info (from nuxt-site-config), page listings with descriptions, and auto-discovered API endpoints (bulk JSONL, MCP).

## Output Files

```bash
.output/public/
├── llms.txt           # Minimal: metadata + page list + endpoints
└── llms-full.txt      # Full markdown content for all pages
```

## Configuration

Add custom sections:

```ts [nuxt.config.ts]
aiReady: { llmsTxt: { sections: [...], notes: '...' } }
```

Full config: [/docs/ai-ready/api/config](/docs/ai-ready/api/config)

## Hook for Advanced Customization

Modify content before writing:

```ts [nuxt.config.ts]
hooks: {
  'ai-ready:llms-txt': (payload) => {
    payload.content += '\n\n## Custom APIs\n\n'
  }
}
```

Hook details: [/docs/ai-ready/api/nuxt-hooks#ai-ready-llms-txt](/docs/ai-ready/api/nuxt-hooks#ai-ready-llms-txt)

## Disabling

```ts [nuxt.config.ts]
aiReady: { bulkRoute: false }  // No llms.txt generation
```

## Dev vs Prod

- **Dev**: Dynamic route handler, on-demand generation
- **Prod**: Static files in `.output/public/`, served directly
