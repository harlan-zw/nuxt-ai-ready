IndexNow is a protocol for instantly notifying search engines when URLs change, enabling faster indexing than waiting for crawlers.

## Supported Search Engines

Submitting to IndexNow notifies all participating engines:

| Engine | Endpoint |
|--------|----------|
| Bing | api.indexnow.org |
| Yandex | yandex.com/indexnow |
| Naver | searchadvisor.naver.com/indexnow |
| Seznam | search.seznam.cz/indexnow |

## Getting Started

### 1. Get an API Key

Generate a key from [Bing Webmaster Tools](https://www.bing.com/indexnow) or create your own (any 32+ character hex string).

### 2. Set the Environment Variable

```bash [.env]
NUXT_AI_READY_INDEXNOW_KEY=your-api-key-here
```

That's it! IndexNow is automatically enabled when the key is present.

## Configuration

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  aiReady: {
    indexNow: {
      enabled: true, // Auto-enabled when key is present
      key: process.env.NUXT_AI_READY_INDEXNOW_KEY, // Or set directly
      host: 'api.indexnow.org' // Optional, default endpoint
    }
  }
})
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | `boolean` | `true` (when key exists) | Enable IndexNow |
| `key` | `string` | `process.env.NUXT_AI_READY_INDEXNOW_KEY` | Your IndexNow API key |
| `host` | `string` | `'api.indexnow.org'` | IndexNow endpoint |

## Automatic Syncing (Scheduled Task)

When both `runtimeSync.cron` and `indexNow` are enabled, IndexNow syncs automatically after each scheduled indexing run:

```
┌─────────────────────────────────────────────────────────────┐
│ Cron triggers ai-ready:index task                           │
│           ↓                                                  │
│ batchIndexPages() → index pending pages                     │
│           ↓                                                  │
│ syncToIndexNow() → submit changed pages to search engines   │
│                  → update indexnow_synced_at                │
└─────────────────────────────────────────────────────────────┘
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  aiReady: {
    runtimeSync: {
      enabled: true,
      cron: '*/5 * * * *' // Every 5 min
    },
    indexNow: {
      key: process.env.NUXT_AI_READY_INDEXNOW_KEY
    }
  }
})
```

For manual integration, use `POST /__ai-ready/indexnow` endpoint.

## How It Works

The module tracks:
- `indexed_at`: When page content was last indexed
- `indexnow_synced_at`: When page was last submitted to IndexNow

Pages need sync when: `indexed = 1` AND `is_error = 0` AND (`indexnow_synced_at IS NULL` OR `indexnow_synced_at < indexed_at`)

## API Endpoints

### Key Verification

Search engines verify your key at `/{key}.txt`:

```bash
GET /your-api-key.txt
# Returns: your-api-key
```

This route is automatically registered.

### Sync Endpoint

Submit pending pages to IndexNow:

```bash
POST /__ai-ready/indexnow?secret=<token>&limit=100
```

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `limit` | `number` | `100` | Max URLs per batch |
| `secret` | `string` | — | Required if `runtimeSync.secret` configured |

**Response:**

```json
{
  "success": true,
  "submitted": 50,
  "remaining": 150,
  "error": null
}
```

### Status Endpoint

Check IndexNow stats:

```bash
GET /__ai-ready/status
```

```json
{
  "total": 100,
  "indexed": 95,
  "pending": 5,
  "indexNow": {
    "pending": 10,
    "totalSubmitted": 500,
    "lastSubmittedAt": 1704067200000,
    "lastError": null
  }
}
```

## Integration with Runtime Sync

Chain IndexNow after polling for complete automation:

```bash
# Index new/changed pages
curl -X POST "https://mysite.com/__ai-ready/poll?secret=xxx"

# Notify search engines of changes
curl -X POST "https://mysite.com/__ai-ready/indexnow?secret=xxx"
```

Or trigger from a scheduled task/cron job:

```ts [server/tasks/sync-search-engines.ts]
import { syncToIndexNow } from '#ai-ready'

export default defineTask({
  meta: { name: 'sync-search-engines' },
  async run({ payload, context }) {
    const result = await syncToIndexNow(context.event, 100)
    return { result: `Submitted ${result.submitted} URLs` }
  }
})
```

## Rate Limits

IndexNow has generous limits:

- Up to 10,000 URLs per request
- Up to 100,000 URLs per day

The module batches URLs efficiently and tracks submission stats.

## Troubleshooting

### Key Verification Fails

Ensure your key route is accessible:

```bash
curl https://yoursite.com/your-api-key.txt
# Should return: your-api-key
```

### Submissions Fail

Check the status endpoint for errors:

```bash
curl https://yoursite.com/__ai-ready/status | jq '.indexNow.lastError'
```

Common errors:
- `403`: Key not found at keyLocation
- `422`: URLs don't match host
- `429`: Rate limited
