title: AI Content Signals
description: Control how AI systems interact with your content through standardized robots.txt directives.
---

Content Signals in `robots.txt` control AI training, output, and indexing permissions. Relies on voluntary compliance - combine with authentication for sensitive content.

## Module Defaults

Nuxt AI Ready auto-configures via [`@nuxtjs/robots`](https://nuxtseo.com/robots) (requires ≥5.6.0):

```ts
{
  contentUsage: ['train-ai=y'],
  contentSignal: ['ai-train=yes', 'search=yes', 'ai-input=yes']
}
```

## Supported Standards

**Content-Usage (IETF aipref-vocab)** - values: `y`/`n`
- `bots` - automated processing
- `train-ai` - foundation model training
- `ai-output` - AI-generated responses
- `search` - search engine indexing

**Content-Signal (Cloudflare/IETF aipref-contentsignals)** - values: `yes`/`no`
- `search` - indexing/snippets
- `ai-input` - RAG, grounding, AI search
- `ai-train` - model training/fine-tuning

## Configuration

Content Signals are configured via [`@nuxtjs/robots`](https://nuxtseo.com/robots), not `aiReady` config.

### Override Defaults

```ts [nuxt.config.ts]
// Block AI training
robots: { groups: [{ contentUsage: { 'train-ai': 'n' } }] }
```

See [@nuxtjs/robots configuration](https://nuxtseo.com/robots/api/config) for all options.

### Path-Specific Rules

```ts [nuxt.config.ts]
// Allow training only for /docs/
robots: { groups: [{ contentUsage: ['train-ai=n', '/docs/ train-ai=y'] }] }
```

::alert{type="info"}
Path-specific rules require string array format.
::

### Bot-Specific Rules

```ts [nuxt.config.ts]
// Block GPTBot
robots: { groups: [{ userAgent: 'GPTBot', disallow: '/' }] }
```

## Syntax

```txt
Content-Usage: <category>=<value>[, <category>=<value>]
Content-Usage: /path/ <category>=<value>

Content-Signal: <category>=<value>[, <category>=<value>]
Content-Signal: /path/ <category>=<value>
```

## Verification

```bash
curl http://localhost:3000/robots.txt
```
