Nuxt AI Ready provides a build-time hook for processing content chunks during prerendering, enabling RAG pipelines, vector databases, and semantic search implementations.

## Build-Time Hook

The `ai-ready:chunk` hook fires for each content chunk during prerendering, allowing you to generate embeddings, populate vector databases, or build custom search indexes.

### Hook Context

```ts
interface ChunkContext {
  chunk: BulkChunk // The chunk data
  route: string // Page route (e.g., '/about')
  title: string // Page title
  description: string // Page description
  headings: Array<Record<string, string>> // Headings structure
}

interface BulkChunk {
  id: string // SHA256-based chunk ID (stable across rebuilds)
  route: string // Page route
  chunkIndex: number // Chunk position (0-based)
  content: string // Markdown content
  headers?: Record<string, string> // Heading hierarchy at chunk level
  loc?: { // Location in HTML
    lines: {
      from: number
      to: number
    }
  }
  title: string // Page title
  description: string // Meta description
}
```

## Usage Examples

### Vector Database Integration

Generate embeddings and populate a vector database:

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  hooks: {
    'ai-ready:chunk': async (context) => {
      const embedding = await generateEmbedding(context.chunk.content)

      await vectorDb.insert({
        id: context.chunk.id,
        embedding,
        content: context.chunk.content,
        metadata: {
          route: context.route,
          title: context.title,
          chunkIndex: context.chunk.chunkIndex,
          headers: context.chunk.headers,
        }
      })
    }
  }
})
```

### Search Index Building

Build a custom search index:

```ts
export default defineNuxtConfig({
  hooks: {
    'ai-ready:chunk': async (context) => {
      await searchIndex.addDocument({
        id: context.chunk.id,
        route: context.route,
        title: context.title,
        content: context.chunk.content,
        headings: context.headings,
      })
    }
  }
})
```

### Multiple Database Sync

Populate multiple systems simultaneously:

```ts
export default defineNuxtConfig({
  hooks: {
    'ai-ready:chunk': async (context) => {
      const embedding = await generateEmbedding(context.chunk.content)

      await Promise.all([
        // Vector database for semantic search
        vectorDb.insert({ id: context.chunk.id, embedding }),
        // Full-text search index
        searchIndex.add({ id: context.chunk.id, content: context.chunk.content }),
        // Analytics/tracking
        analytics.track('chunk_indexed', { route: context.route }),
      ])
    }
  }
})
```

## Chunk ID Stability

Chunk IDs are deterministic SHA256 hashes based on route and index, ensuring stability across rebuilds:

```ts
// Generated by nuxt-ai-ready
function generateVectorId(route: string, chunkIndex: number): string {
  const hash = createHash('sha256')
    .update(route)
    .digest('hex')
    .substring(0, 8)
  return `${hash}-${chunkIndex}`
}
```

**Benefits:**
- Same route + index = same ID across rebuilds
- Enables efficient incremental updates
- No need to regenerate embeddings for unchanged content

## Configuration

### Chunk Size

Control chunk size for optimal embedding:

```ts
export default defineNuxtConfig({
  aiReady: {
    mdreamOptions: {
      chunkSize: 512, // Tokens per chunk (default: 256)
    }
  }
})
```

### Route Filtering

Exclude routes from processing:

```ts
export default defineNuxtConfig({
  nitro: {
    prerender: {
      ignore: ['/admin/**', '/api/**']
    }
  }
})
```

## Best Practices

1. **Batch Operations**: Collect chunks and batch insert to vector DB for better performance
2. **Error Handling**: Wrap hook logic in try/catch to prevent build failures
3. **Incremental Updates**: Use stable chunk IDs to only update changed content
4. **Metadata**: Store relevant metadata (route, title, headers) alongside embeddings for filtering

See [API Reference](/docs/ai-ready/api/nuxt-hooks#ai-ready-chunk) for complete hook documentation.
