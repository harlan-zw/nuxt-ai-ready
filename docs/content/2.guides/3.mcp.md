Nuxt AI Ready includes built-in support for [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) when
the [`@nuxtjs/mcp-toolkit`](https://github.com/nuxt-modules/mcp-toolkit) is installed.

Your site will expose MCP tools and resources to AI agents like [Claude](https://claude.ai/) based on your indexed content.

## Installation

```bash
npx nuxi module add @nuxtjs/mcp-toolkit
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    'nuxt-ai-ready',
    '@nuxtjs/mcp-toolkit',
  ],
})
```

See [@nuxtjs/mcp-toolkit docs](https://github.com/nuxt-modules/mcp) for server configuration options.

## Connection

Add to Claude Desktop config:

```json
{
  "mcpServers": {
    "my-site": {
      "command": "npx",
      "args": ["-y", "@nuxtjs/mcp-client", "https://example.com/mcp"]
    }
  }
}
```

## Built-in Tools

### `list_pages`

Lists all available pages in [TOON format](https://toonformat.dev) - a token-efficient encoding designed for LLM input. Choose between minimal page-level metadata or individual content chunks.

**Parameters:**
- `mode` (enum): `'minimal'` (default) or `'chunks'`
  - `minimal`: Returns page-level metadata (route, title, description, headings, id, chunkIds)
  - `chunks`: Returns individual content chunks with chunk-level metadata

**Response:** TOON-encoded data (more token-efficient than JSON)

Minimal mode response example:
```toon
pages[2]{route,title,description,id}:
  /docs/getting-started,Getting Started,Quick start guide...,abc123
  /docs/installation,Installation,How to install...,def456
```

Chunks mode response example:
```toon
chunks[3]{id,route,chunkIndex,content,title}:
  abc123-0,/docs/getting-started,0,"# Getting Started...",Getting Started
  abc123-1,/docs/getting-started,1,"## Installation...",Getting Started
  def456-0,/docs/installation,0,"# Installation...",Installation
```

TOON combines YAML-style indentation with CSV-style tabular data for uniform arrays, saving significant tokens compared to JSON.

## Built-in Resources

Resources provide static data access to AI agents. Unlike tools, resources don't accept parameters.

### `resource://nuxt-ai-ready/pages`

Complete list of all pages with metadata (route, title, description, headings) in TOON format.

**Format:** [TOON](https://toonformat.dev) (Token-Oriented Object Notation) - a token-efficient encoding of JSON designed for LLM input. Contains page-level metadata as a compact array.

### `resource://nuxt-ai-ready/pages-chunks`

Individual content chunks from all pages in TOON format.

**Format:** [TOON](https://toonformat.dev) - contains chunk-level metadata (id, route, chunkIndex, content, title, description) in a token-optimized format

## Common Workflows

### Browse All Pages
1. Use `list_pages` tool with `mode: 'minimal'` to get page list with metadata
2. Review routes, titles, descriptions
3. Use `list_pages` with `mode: 'chunks'` to get individual content chunks

### Bulk Processing
1. Access `resource://nuxt-ai-ready/pages` for page-level metadata in TOON format
2. Or access `resource://nuxt-ai-ready/pages-chunks` for individual content chunks in TOON format
3. Process all pages/chunks - TOON is natively readable by LLMs and more token-efficient than JSON
