```ts [nuxt.config.ts]
export default defineNuxtConfig({
  aiReady: {
    // options
  }
})
```

## `enabled`

- **Type:** `boolean`{lang="ts"}
- **Default:** `true`{lang="ts"}

```ts
aiReady: {
  enabled: process.env.NODE_ENV === 'production'
}
```

## `debug`

- **Type:** `boolean`{lang="ts"}
- **Default:** `false`{lang="ts"}

Enable debug logging for module operations.

## `mdreamOptions`

- **Type:** `HTMLToMarkdownOptions & { preset?: 'minimal' }`{lang="ts"}
- **Default:** `{ preset: 'minimal' }`{lang="ts"}

Configure [mdream](https://github.com/harlan-zw/mdream) HTML-to-markdown conversion.

```ts
export default defineNuxtConfig({
  aiReady: {
    mdreamOptions: {
      preset: 'minimal'
    }
  }
})
```

## `markdownCacheHeaders`

- **Type:** `{ maxAge?: number, swr?: boolean }`{lang="ts"}
- **Default:** `{ maxAge: 3600, swr: true }`{lang="ts"}

Cache settings for runtime markdown endpoints.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `maxAge` | `number` | `3600` | Cache duration in seconds |
| `swr` | `boolean` | `true` | Stale-while-revalidate |

## `llmsTxt`

- **Type:** `LlmsTxtConfig`{lang="ts"}
- **Default:** Auto-generated

Configure llms.txt generation.

```ts
export default defineNuxtConfig({
  aiReady: {
    llmsTxt: {
      sections: [
        {
          title: 'API Reference',
          links: [
            { title: 'REST API', href: '/docs/api', description: 'API docs' }
          ]
        }
      ],
      notes: 'Built with Nuxt AI Ready'
    }
  }
})
```

**LlmsTxtSection:**

| Property | Type | Description |
|----------|------|-------------|
| `title` | `string` | Section title |
| `description` | `string \| string[]` | Section description |
| `links` | `{ title, href, description? }[]` | Links in section |

**LlmsTxtConfig:**

| Property | Type | Description |
|----------|------|-------------|
| `sections` | `LlmsTxtSection[]` | Custom sections |
| `notes` | `string \| string[]` | Notes at end |

## `contentSignal`

- **Type:** `false | { aiTrain?: boolean, search?: boolean, aiInput?: boolean }`{lang="ts"}
- **Default:** `false`{lang="ts"}

Content Signal directives for robots.txt. See [Content Signals guide](/docs/ai-ready/guides/content-signals).

```ts
export default defineNuxtConfig({
  aiReady: {
    contentSignal: {
      aiTrain: false, // Block training
      search: true, // Allow search indexing
      aiInput: true, // Allow RAG/grounding
    }
  }
})
```

## `mcp`

- **Type:** `{ tools?: boolean, resources?: boolean }`{lang="ts"}
- **Default:** `{ tools: true, resources: true }`{lang="ts"}

Control MCP features when `@nuxtjs/mcp-toolkit` installed.

```ts
export default defineNuxtConfig({
  aiReady: {
    mcp: {
      tools: true, // list_pages, search_pages
      resources: true, // pages resource
    }
  }
})
```

See [MCP guide](/docs/ai-ready/guides/mcp) for tool/resource details.

## `cacheMaxAgeSeconds`

- **Type:** `number`{lang="ts"}
- **Default:** `600`{lang="ts"} (10 minutes)

Cache duration for llms.txt route handlers. Uses stale-while-revalidate.

```ts
export default defineNuxtConfig({
  aiReady: {
    cacheMaxAgeSeconds: 3600 // 1 hour
  }
})
```

## `ttl`

- **Type:** `number`{lang="ts"}
- **Default:** `0`{lang="ts"}

TTL (time-to-live) for re-indexing pages in seconds. Pages are automatically indexed on visit; this controls how often to refresh stale entries.

| Value | Behavior |
|-------|----------|
| `0` | Never re-index (rely on prerendered data) |
| `3600` | Re-index if page was indexed >1 hour ago |
| `86400` | Re-index daily |

```ts
export default defineNuxtConfig({
  aiReady: {
    ttl: 3600 // re-index after 1 hour
  }
})
```

**How indexing works:**

1. On cold start, prerendered data is restored from a compressed dump
2. As pages are visited, they're indexed in the background via `waitUntil`
3. MCP tools use SQLite FTS5 for full-text search
4. Data persists in SQLite (file-based or in-memory depending on platform)

## `database`

- **Type:** `{ type?: string, filename?: string, bindingName?: string, url?: string, authToken?: string }`{lang="ts"}
- **Default:** `{ type: 'sqlite', filename: '.data/ai-ready/pages.db' }`{lang="ts"}

Configure the SQLite database for page storage. The module auto-detects the best connector based on runtime.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `type` | `'sqlite' \| 'd1' \| 'libsql'` | `'sqlite'` | Database type |
| `filename` | `string` | `'.data/ai-ready/pages.db'` | SQLite file path |
| `bindingName` | `string` | `'AI_READY_DB'` | D1 binding name |
| `url` | `string` | — | LibSQL/Turso URL |
| `authToken` | `string` | — | LibSQL/Turso auth token |

::code-group
```ts [Default (SQLite)]
export default defineNuxtConfig({
  aiReady: {
    database: {
      filename: '.data/ai-ready/pages.db'
    }
  }
})
```

```ts [Cloudflare D1]
export default defineNuxtConfig({
  aiReady: {
    database: {
      type: 'd1',
      bindingName: 'AI_READY_DB'
    }
  }
})
```

```ts [Turso/LibSQL]
export default defineNuxtConfig({
  aiReady: {
    database: {
      type: 'libsql',
      url: process.env.TURSO_URL,
      authToken: process.env.TURSO_AUTH_TOKEN
    }
  }
})
```
::

**SQLite connector auto-detection:**

| Runtime | Connector |
|---------|-----------|
| Bun | `bun:sqlite` |
| Node.js 22.5+ | `node:sqlite` |
| Node.js <22.5 | `better-sqlite3` |

For serverless deployments, the module generates a compressed dump at build time that's restored on cold start.
