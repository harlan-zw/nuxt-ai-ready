```ts [nuxt.config.ts]
export default defineNuxtConfig({
  aiReady: {
    // options
  }
})
```

## `enabled`

- **Type:** `boolean`{lang="ts"}
- **Default:** `true`{lang="ts"}

```ts
aiReady: {
  enabled: process.env.NODE_ENV === 'production'
}
```

## `debug`

- **Type:** `boolean`{lang="ts"}
- **Default:** `false`{lang="ts"}

Enable debug logging for module operations.

## `mdreamOptions`

- **Type:** `HTMLToMarkdownOptions & { preset?: 'minimal' }`{lang="ts"}
- **Default:** `{ preset: 'minimal' }`{lang="ts"}

Configure [mdream](https://github.com/harlan-zw/mdream) HTML-to-markdown conversion.

```ts
export default defineNuxtConfig({
  aiReady: {
    mdreamOptions: {
      preset: 'minimal'
    }
  }
})
```

## `markdownCacheHeaders`

- **Type:** `{ maxAge?: number, swr?: boolean }`{lang="ts"}
- **Default:** `{ maxAge: 3600, swr: true }`{lang="ts"}

Cache settings for runtime markdown endpoints.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `maxAge` | `number` | `3600` | Cache duration in seconds |
| `swr` | `boolean` | `true` | Stale-while-revalidate |

## `llmsTxt`

- **Type:** `LlmsTxtConfig`{lang="ts"}
- **Default:** Auto-generated

Configure llms.txt generation.

```ts
export default defineNuxtConfig({
  aiReady: {
    llmsTxt: {
      sections: [
        {
          title: 'API Reference',
          links: [
            { title: 'REST API', href: '/docs/api', description: 'API docs' }
          ]
        }
      ],
      notes: 'Built with Nuxt AI Ready'
    }
  }
})
```

**LlmsTxtSection:**

| Property | Type | Description |
|----------|------|-------------|
| `title` | `string` | Section title |
| `description` | `string \| string[]` | Section description |
| `links` | `{ title, href, description? }[]` | Links in section |

**LlmsTxtConfig:**

| Property | Type | Description |
|----------|------|-------------|
| `sections` | `LlmsTxtSection[]` | Custom sections |
| `notes` | `string \| string[]` | Notes at end |

## `contentSignal`

- **Type:** `false | { aiTrain?: boolean, search?: boolean, aiInput?: boolean }`{lang="ts"}
- **Default:** `false`{lang="ts"}

Content Signal directives for robots.txt. See [Content Signals guide](/docs/ai-ready/guides/content-signals).

```ts
export default defineNuxtConfig({
  aiReady: {
    contentSignal: {
      aiTrain: false, // Block training
      search: true, // Allow search indexing
      aiInput: true, // Allow RAG/grounding
    }
  }
})
```

## `mcp`

- **Type:** `{ tools?: boolean, resources?: boolean }`{lang="ts"}
- **Default:** `{ tools: true, resources: true }`{lang="ts"}

Control MCP features when `@nuxtjs/mcp-toolkit` installed.

```ts
export default defineNuxtConfig({
  aiReady: {
    mcp: {
      tools: true, // list_pages, search_pages
      resources: true, // pages resource
    }
  }
})
```

See [MCP guide](/docs/ai-ready/guides/mcp) for tool/resource details.

## `cacheMaxAgeSeconds`

- **Type:** `number`{lang="ts"}
- **Default:** `600`{lang="ts"} (10 minutes)

Cache duration for llms.txt route handlers. Uses stale-while-revalidate.

```ts
export default defineNuxtConfig({
  aiReady: {
    cacheMaxAgeSeconds: 3600 // 1 hour
  }
})
```

## `pruneTtl`

- **Type:** `number`{lang="ts"}
- **Default:** `0`{lang="ts"} (never prune)

TTL for pruning stale routes not seen in sitemap (in seconds). Routes not seen in sitemap for longer than this threshold are removed.

| Value | Behavior |
|-------|----------|
| `0` | Never prune routes |
| `604800` | Prune routes not seen for 7 days |
| `2592000` | Prune routes not seen for 30 days |

```ts
export default defineNuxtConfig({
  aiReady: {
    pruneTtl: 604800 // prune after 7 days
  }
})
```

## `database`

- **Type:** `{ type?: string, filename?: string, bindingName?: string, url?: string, authToken?: string }`{lang="ts"}
- **Default:** `{ type: 'sqlite', filename: '.data/ai-ready/pages.db' }`{lang="ts"}

Configure the SQLite database for page storage. The module auto-detects the best connector based on runtime.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `type` | `'sqlite' \| 'd1' \| 'libsql'` | `'sqlite'` | Database type |
| `filename` | `string` | `'.data/ai-ready/pages.db'` | SQLite file path |
| `bindingName` | `string` | `'AI_READY_DB'` | D1 binding name |
| `url` | `string` | — | LibSQL/Turso URL |
| `authToken` | `string` | — | LibSQL/Turso auth token |

::code-group
```ts [Default (SQLite)]
export default defineNuxtConfig({
  aiReady: {
    database: {
      filename: '.data/ai-ready/pages.db'
    }
  }
})
```

```ts [Cloudflare D1]
export default defineNuxtConfig({
  aiReady: {
    database: {
      type: 'd1',
      bindingName: 'AI_READY_DB'
    }
  }
})
```

```ts [Turso/LibSQL]
export default defineNuxtConfig({
  aiReady: {
    database: {
      type: 'libsql',
      url: process.env.TURSO_URL,
      authToken: process.env.TURSO_AUTH_TOKEN
    }
  }
})
```
::

**SQLite connector auto-detection:**

| Runtime | Connector |
|---------|-----------|
| Bun | `bun:sqlite` |
| Node.js 22.5+ | `node:sqlite` |
| Node.js <22.5 | `better-sqlite3` |

For serverless deployments, the module generates a compressed dump at build time that's restored on cold start.

**Table naming:**

All tables are prefixed with `ai_ready_` to avoid conflicts with existing database tables:
- `ai_ready_pages` - Main pages table
- `ai_ready_pages_fts` - FTS5 full-text search index
- `_ai_ready_info` - Schema version tracking

This allows safe use with existing D1/LibSQL databases without risk of overwriting user tables.

## `runtimeSync`

- **Type:** `{ enabled?: boolean, ttl?: number, sitemapTtl?: number, batchSize?: number, cron?: string, secret?: string }`{lang="ts"}
- **Default:** `{ enabled: false }`{lang="ts"}

**Opt-in runtime sync for dynamic content sites.** Most sites don't need this—prerendering handles page indexing automatically.

Enable only if your site has frequently changing content that can't be prerendered.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | `boolean` | `false` | Enable runtime sync |
| `ttl` | `number` | `3600` | Re-index pages older than this (seconds) |
| `sitemapTtl` | `number` | `3600` | Refresh sitemap routes (seconds) |
| `batchSize` | `number` | `20` | Pages per batch (max: 50) |
| `cron` | `string` | — | Cron expression for scheduled indexing |
| `secret` | `string` | — | Auth token for `/__ai-ready/poll` endpoint |

```ts
export default defineNuxtConfig({
  aiReady: {
    runtimeSync: {
      enabled: true,
      ttl: 3600,
      sitemapTtl: 3600,
      batchSize: 20,
      secret: process.env.AI_READY_SECRET,
      cron: '*/5 * * * *' // enable scheduled indexing
    }
  }
})
```

**When enabled:**

- `GET /__ai-ready/status` - Check indexing progress
- `POST /__ai-ready/poll?secret=<token>` - Trigger batch indexing

See [Runtime Sync guide](/docs/ai-ready/guides/runtime-indexing) for details.

## `indexNow`

- **Type:** `{ enabled?: boolean, key?: string, host?: string }`{lang="ts"}
- **Default:** Auto-enabled when `NUXT_AI_READY_INDEXNOW_KEY` env var is set

Instantly notify search engines (Bing, Yandex, Naver, Seznam) when pages change.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | `boolean` | `true` (when key exists) | Enable IndexNow |
| `key` | `string` | `process.env.NUXT_AI_READY_INDEXNOW_KEY` | Your IndexNow API key |
| `host` | `string` | `'api.indexnow.org'` | IndexNow endpoint |

**Minimal setup (just set env var):**

```bash [.env]
NUXT_AI_READY_INDEXNOW_KEY=your-api-key-here
```

**Explicit configuration:**

```ts
export default defineNuxtConfig({
  aiReady: {
    indexNow: {
      enabled: true,
      key: process.env.INDEXNOW_KEY,
      host: 'api.indexnow.org'
    }
  }
})
```

When enabled:
- Key verification route registered at `/{key}.txt`
- Automatic sync after scheduled indexing (when `runtimeSync.cron` set)
- Manual sync via `POST /__ai-ready/indexnow`
- Stats included in `/__ai-ready/status`

See [IndexNow guide](/docs/ai-ready/guides/indexnow) for details.
