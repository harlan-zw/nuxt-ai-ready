## MCP Integration (Optional)

Connect AI agents to your site via [Model Context Protocol](https://modelcontextprotocol.io/):

```bash
npx nuxi module add @nuxtjs/mcp-toolkit
```

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    'nuxt-ai-ready',
    '@nuxtjs/mcp-toolkit',
  ],
})
```

Add to Claude Desktop config (`~/Library/Application Support/Claude/claude_desktop_config.json` on macOS):

```json
{
  "mcpServers": {
    "my-site": {
      "command": "npx",
      "args": ["-y", "@nuxtjs/mcp-client", "http://localhost:3000/mcp"]
    }
  }
}
```

See the [MCP guide](/docs/ai-ready/guides/mcp) for tools, resources, and production setup.

## Verifying Installation

Zero-config module. No additional setup needed.

**Dev mode:**
- Visit any route with `.md` suffix (e.g., `/about.md`) to see markdown output

**After production build:**
- `/robots.txt` shows [Content Signals](/docs/ai-ready/guides/content-signals)
- `/llms.txt` and `/llms-full.txt` generated with page content
- MCP tools return page data with fuzzy search

## Rendering Mode Support

Works best with prerendering. Feature availability by mode:

| Feature | SSG | Hybrid | SSR-only | SPA |
|---------|-----|--------|----------|-----|
| llms.txt with titles | ✅ | ✅ Prerendered only | ❌ Routes only | ❌ |
| llms-full.txt | ✅ | ✅ Prerendered only | ❌ Placeholder | ❌ |
| Runtime `.md` routes | ✅ | ✅ | ✅ | ❌ |
| MCP tools/resources | ✅ | ✅ Partial | ❌ Empty | ❌ |
| `ai-ready:page:markdown` hook | ✅ | ✅ Prerendered only | ❌ Never fires | ❌ |

**SSG**: Full support. All pages processed during `nuxi generate`{lang="bash"}.

**Hybrid**: Page data only for prerendered pages. llms.txt splits into "Prerendered Pages" vs "Other Pages".

**SSR-only**: No page data generated. llms.txt works from sitemap but without titles. Runtime `.md` routes work.

**SPA**: Not supported. Workaround: enable `nitro.prerender.routes`{lang="ts"} for specific pages.
